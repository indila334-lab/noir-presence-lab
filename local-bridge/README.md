# Severin local bridge

Локальный мост без внешнего API.

## Зачем

GitHub Pages умеет показать страницу, записать микрофон и проиграть звук, но не умеет быть сервером разговора.

Этот мост делает первый настоящий канал:

```text
TV/cube browser -> local Wi-Fi server -> saved voice files
local answer file -> TV/cube browser -> speaker/Alice/колонка
```

## Запуск

На ноутбуке / мини-ПК / будущем кубике в этой папке:

```bash
python server.py
```

Сервер поднимется на порту `8787`.

На ТВ-приставке открыть:

```text
http://LOCAL_IP:8787/
```

Где `LOCAL_IP` — IP устройства, на котором запущен сервер, например:

```text
http://192.168.1.34:8787/
```

## Что уже работает

- Запись голоса с браузера.
- Отправка записи на локальный сервер.
- Сохранение записей в `local-bridge/inbox/`.
- Проигрывание ответного файла из `local-bridge/outbox/answer.wav` или `answer.mp3`.

## Что делать вручную без API

1. ТВ записывает голос Марина.
2. Сервер сохраняет файл в `inbox/`.
3. Человек/локальная программа готовит ответный файл `answer.wav` или `answer.mp3`.
4. Кладёт его в `outbox/`.
5. ТВ нажимает «Проверить ответный файл» и слышит Северина.
