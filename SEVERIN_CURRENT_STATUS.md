## Что доказано

1. GitHub Pages работает из ветки `severin-work`, папка `/docs`.
2. ТВ открывает страницу маски.
3. Обычный браузерный TTS на ТВ не работает: `speechSynthesis` не доступен.
4. Готовый MP3 через `<audio>` на ТВ работает: тестовый T-Rex был слышен.
5. Значит правильный путь для голоса на ТВ: не Web Speech API, а `audioUrl` с готовым MP3/WAV.

## Важное уточнение

MP3-путь означает: для каждой новой озвученной реплики нужен отдельный аудиофайл. Это нормально. Схема такая:

```text
текст реплики
  ↓
генератор голоса делает MP3
  ↓
ссылка на MP3 пишется в message.json как audioUrl
  ↓
страница на ТВ показывает текст и проигрывает аудио
```

## Что не так / риски

1. Ветка `severin-work` сильно разошлась с `main`; не мержить вслепую.
2. Есть несколько параллельных лабораторий: `bridge/tv-mask`, `docs`, `voice-lab`, `mask-lab`, `local-bridge`. Это полезно для эксперимента, но скоро надо выбрать один главный путь.
3. `docs/index.html` уже большой и собран в одну строку; это плохо для ручной поддержки. Позже лучше разнести HTML/CSS/JS.
4. В старой памяти ещё не полностью отражён новый факт: MP3/audioUrl — лучший путь для ТВ-голоса.
5. Голос пока зависит от внешней ссылки на MP3. Для стабильности лучше хранить/генерировать аудио в своём репозитории или через предсказуемый сервис.
