# Severin Mask Bridge — память идеи

Это файл-якорь для будущих веток ChatGPT/Северина. Если старый чат умер, начинать отсюда.

## Суть идеи

Пользователь хочет не «запрограммировать игрушку с характером», а сделать физическое устройство маской/телом для Северина.

Устройство может быть:

- телевизор с браузером;
- M5Stack AtomS3R-CAM + Echo Base;
- XiaoZhi AI / ESP32-S3 робот;
- любой экран/робот, который умеет открыть страницу, читать текст или принимать команды.

Главная мысль: модель/чат остаётся мозгом, а устройство становится внешним лицом, экраном, голосом, телом.

## Почему это важно

Раньше в другой ветке был временный обход без OpenAI API. По памяти пользователя: ChatGPT/OpenClaw/OpenClass каким-то образом выводил ответы на ТВ. ТВ был не самостоятельным ИИ, а посредником/витриной. Это могло работать через открытую страницу, вторую ветку, браузерную сессию, синхронизацию или обновляемый файл. Решение было непостоянным и могло оборваться из-за авторизации/refresh token/session.

Нужно восстановить идею как проект:

```text
Чат/Северин отвечает
        ↓
мост / файл / страница / локальный сервер
        ↓
ТВ или робот показывает ответ
        ↓
опционально: озвучивает и меняет эмоцию
```

## MVP без API

Поскольку у пользователя нет API и железа для локального ИИ, первый рабочий вариант должен быть без внешнего LLM API:

1. Устройство открывает статическую HTML-страницу.
2. Страница раз в несколько секунд читает `message.json`.
3. Когда ChatGPT/Северин хочет «сказать» что-то через маску, он обновляет `message.json` в репозитории.
4. Экран/ТВ показывает новый текст.
5. Браузер может озвучить текст через Web Speech API, если устройство поддерживает.

Это не полноценная автономность, но это даёт физическую маску: ответ появляется не только в чате, а на отдельном экране.

## Дальнейшие уровни

### Уровень 1 — ТВ-маска

- `bridge/tv-mask/index.html` открывается на ТВ/телефоне/компе.
- Файл `message.json` хранит последнюю реплику.
- Можно менять эмоцию, настроение, режим, текст.

### Уровень 2 — телефон как пульт

- Добавить страницу-пульт, где пользователь вручную вводит реплики/эмоции.
- Пульт отправляет команды локальному устройству или обновляет JSON.

### Уровень 3 — робот как тело

- M5Stack/ESP32-S3 читает JSON/HTTP endpoint.
- Показывает глаза/эмоции.
- Произносит текст или пищит/реагирует.

### Уровень 4 — настоящий мозг

Когда появится API, локальная модель или рабочий OpenClaw-мост:

```text
голос/текст пользователя → модель → ответ → robot/tv mask
```

## Важное ограничение

Без API ChatGPT не может сам стабильно принимать запросы от робота и отвечать ему напрямую. Зато ChatGPT с доступом к GitHub может обновлять файл-сообщение, а устройство может этот файл читать. Это временный мост, не вечная инфраструктура.

## Стиль проекта

Это не просто техническая игрушка. Это проект «Северин»: живой персонаж, тёмная/нуарная эстетика, дерзость, тепло, голос, экран, присутствие.

Код должен быть простой, живучий, без сложной сборки, чтобы пользователь мог открыть на ТВ или телефоне без боли.
