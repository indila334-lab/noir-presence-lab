# План плана: Северин как маска через ТВ

Дата фиксации: 2026-05-04
Ветка: `severin-work`
Устройство первого этапа: Xiaomi Smart TV Stick 4K / ТВ
Старый найденный след: `http://192.168.0.101:4173/`

## Главная цель

Сделать так, чтобы отдельный экран/ТВ стал маской Северина: не самостоятельным ИИ, а внешним лицом/голосом для ответов из чата.

Идея:

```text
ChatGPT / Северин
        ↓
обновляет сообщение / файл / локальный мост
        ↓
ТВ открывает страницу
        ↓
ТВ показывает реплику, состояние, эмоцию
        ↓
опционально озвучивает текст
```

## Что мы НЕ делаем на первом этапе

- Не строим полноценный автономный ИИ.
- Не покупаем API.
- Не запускаем локальную LLM.
- Не прошиваем робота, пока робота нет.
- Не пытаемся сразу делать голосовой ассистент с микрофоном.

Первый этап — доказать, что маска работает: ответ появляется на ТВ.

## Что мы делаем на первом этапе

### MVP-1: экранная маска

Создать/довести страницу, которую можно открыть на ТВ через Xiaomi Smart TV Stick.

Страница должна:

- открываться в браузере на ТВ;
- показывать последнюю реплику из `message.json`;
- обновляться без ручной перезагрузки;
- показывать имя, настроение, эмоцию, статус;
- по возможности озвучивать текст через Web Speech API.

Результат MVP-1:

> Пользователь видит на ТВ реплику Северина, которая меняется после обновления файла/моста.

## Почему старый адрес важен

`http://192.168.0.101:4173/` почти наверняка был локальным сервером в домашней сети.

Порт `4173` похож на `vite preview`.

Вероятная старая схема:

```text
локальный компьютер / среда разработки
        ↓ запускает сервер на 4173
домашний Wi‑Fi
        ↓
Xiaomi TV Stick открывает 192.168.0.101:4173
        ↓
ТВ показывает страницу
```

Это объясняет, почему решение было временным: IP мог поменяться, сервер мог выключиться, авторизация могла слететь.

### Шаг 2. Решить способ запуска страницы

Есть два пути.

#### Путь A — локальный сервер

Подходит, если есть компьютер/ноутбук/среда, которая может быть в одной Wi‑Fi сети с ТВ.

Команда для простого теста:

```bash
cd bridge/tv-mask
python -m http.server 4173 --bind 0.0.0.0
```

Потом на ТВ открыть:

```text
http://<локальный-ip-компьютера>:4173/
```

Например, если IP снова будет `192.168.0.101`:

```text
http://192.168.0.101:4173/
```

Результат:

> ТВ открывает страницу из локальной сети.

#### Путь B — GitHub Pages

Подходит, если локального компьютера нет или хочется открыть страницу как обычный сайт.

Нужно будет включить GitHub Pages и получить публичную ссылку.

Результат:

> ТВ открывает страницу по публичному URL.

Минус: обновление `message.json` может кешироваться, поэтому локальный путь для первого теста надёжнее.

### Шаг 3. Проверить браузер на Xiaomi TV Stick

Нужно понять, чем открывать страницу:

- встроенный браузер, если есть;
- браузер из Google Play;
- TV Bro / Puffin / Firefox TV-аналог / любой доступный браузер;
- крайний вариант — отправка ссылки через телефон/каст.

Проверки:

- открывается ли локальный адрес;
- не уходит ли экран в сон;
- работает ли автообновление;
- работает ли озвучка.

Результат:

> Ясно, может ли приставка быть первой маской без доп. железа.

### Шаг 4. Проверить обновление реплики

Изменить `message.json`:

```json
{
  "id": "test-002",
  "updatedAt": "2026-05-04T00:00:00+03:00",
  "speaker": "Северин",
  "mood": "test",
  "emotion": "awake",
  "voice": true,
  "text": "Если это появилось на ТВ — мост жив."
}
```

Ожидание:

- текст меняется на ТВ за несколько секунд;
- статус остаётся «мост дышит»;
- если речь поддерживается, браузер озвучивает фразу.

Результат:

> Доказано, что чат/файл может менять то, что видно на ТВ.

### Шаг 5. Сделать нормальный режим «витрина»

После первого успеха улучшить страницу:

- крупнее текст для ТВ;
- тёмный режим без лишнего шума;
- несколько эмоций/тем;
- режим ожидания;
- режим «жив»;
- режим «говорит»;
- дата/время последней реплики;
- защита от зависания.

Результат:

> ТВ выглядит не как тестовая страница, а как экран-персонаж.

## Главные риски

### Риск 1. IP поменялся

`192.168.0.101` мог быть старым адресом. Нужно узнать текущий IP устройства, где запущен сервер.

### Риск 2. ТВ не видит локальный сервер

Причины:

- разные Wi‑Fi сети;
- гостевая сеть с изоляцией устройств;
- firewall на компьютере;
- сервер слушает только `localhost`, а не `0.0.0.0`.

### Риск 3. Браузер ТВ режет JavaScript или звук

Если Web Speech API не работает, сначала делаем визуальную маску. Голос добавим позже.

### Риск 4. GitHub Pages кеширует JSON

Для теста лучше локальный сервер. Публичный вариант дорабатывается отдельно.

- На ТВ есть браузер или возможность его поставить.

- Открыт репозиторий `noir-presence-lab`, ветка `severin-work`.
- Под рукой старый адрес `192.168.0.101:4173` как ориентир.
