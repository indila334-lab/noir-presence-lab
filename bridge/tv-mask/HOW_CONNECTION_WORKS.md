# Как Северин подключается к странице

Коротко: напрямую ChatGPT не «заходит» в страницу как человек и не управляет ТВ. Связь делается через общий объект, который видят обе стороны.

В нашем MVP общий объект — файл `message.json` в GitHub.

## Схема через GitHub

```text
1. Северин в чате получает от пользователя реплику
        ↓
2. Северин через GitHub-connector обновляет bridge/tv-mask/message.json
        ↓
3. Страница index.html на ТВ каждые несколько секунд читает message.json
        ↓
4. Если id изменился — страница показывает новый текст и пытается озвучить
```

То есть GitHub — это не мозг. Это доска объявлений / почтовый ящик между чатом и экраном.

## Почему это легче, чем старый мост

Старый мост мог держаться на локальном сервере, браузерной сессии, OpenClaw/OpenClass и временных токенах. Поэтому он мог умереть.

GitHub проще для первого теста:

- ChatGPT уже имеет доступ к репозиторию;
- можно обновлять файл из чата;
- ТВ может открыть страницу и читать файл;
- не нужен OpenAI API;
- не нужен локальный ИИ;
- не нужно сразу прошивать робота.

## Важное ограничение

Страница на ТВ сама не умеет задавать вопросы ChatGPT. Она только читает то, что ей положили в `message.json`.

Это значит:

```text
Пользователь пишет Северину в чате → Северин обновляет JSON → ТВ показывает ответ
```

Но не так:

```text
Пользователь говорит в ТВ → ТВ само спрашивает Северина → Северин сам отвечает
```

Второй вариант появится позже, когда будет API, локальный сервер, OpenClaw-мост или другой канал ввода.

## Два режима запуска

### Режим A: GitHub Pages

ТВ открывает публичную ссылку на страницу из репозитория.

Плюс: не нужен включённый домашний компьютер.
Минус: GitHub может кешировать файлы, обновления могут идти не мгновенно.

### Режим B: локальный сервер 4173

На компьютере/телефоне/среде запускается сервер:

```bash
cd bridge/tv-mask
python -m http.server 4173 --bind 0.0.0.0
```

ТВ открывает:

```text
http://<локальный-ip>:4173/
```

Плюс: похоже на старую схему `192.168.0.101:4173`, меньше проблем с кешем.
Минус: устройство с сервером должно быть включено и быть в той же Wi‑Fi сети.
