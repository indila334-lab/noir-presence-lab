# Локальная страница маски: `http://192.168.0.101:4173/`

Пользователь вспомнил, что раньше страница моста/маски открывалась по адресу:

```text
http://192.168.0.101:4173/
```

## Что это, скорее всего

Это был не публичный сайт, а локальный сервер в домашней сети.

Порт `4173` очень похож на стандартный порт `vite preview`.

Вероятная схема:

```text
компьютер / среда разработки
  запускает Vite dev/preview server
        ↓
локальный адрес в Wi‑Fi сети: 192.168.0.101:4173
        ↓
Xiaomi TV Stick / ТВ открывает этот адрес в браузере
        ↓
ТВ показывает страницу-маску
```

## Почему страница работала только дома

Адрес `192.168.0.101` — локальный LAN IP. Он доступен только устройствам в той же Wi‑Fi/домашней сети.

Если сервер выключен, IP изменился, компьютер уснул или приставка в другой сети — страница не откроется.

## Как повторить

В будущей реализации можно поднять локальную страницу так:

```bash
cd bridge/tv-mask
python -m http.server 4173 --bind 0.0.0.0
```

И открыть на ТВ:

```text
http://<локальный-ip-компьютера>:4173/
```

Если используется Vite-проект, то запуск может быть похож на:

```bash
npm run dev -- --host 0.0.0.0
```

или:

```bash
npm run preview -- --host 0.0.0.0 --port 4173
```

## Важная догадка

Старый обход, вероятно, был не через OpenAI API, а через локальную веб-страницу, поднятую в домашней сети. ТВ/приставка была экраном, который заходил на страницу `192.168.0.101:4173`. Мозг оставался в чате/браузерной сессии/локальном мосте, а ТВ было маской.

## Что проверить

1. Найти устройство с IP `192.168.0.101` в домашней сети.
2. Проверить, не был ли это компьютер/ноутбук/телефон, где запускался Vite или локальный сервер.
3. На этом устройстве проверить команды/папки проекта, где мог быть запуск `npm`, `vite`, `preview`, `localhost:4173`.
4. Для Xiaomi TV Stick использовать браузер и открывать локальный IP сервера.
